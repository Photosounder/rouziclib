== Integration model

rouziclib is normally embedded into an application rather than built as a separately installed binary. The application supplies a small configuration header, includes the public header from that configuration header, and includes the implementation once.

The usual arrangement is:

[%header,cols="1,3"]
|===
|File |Role
|`rl.h` |Application-specific feature defines followed by `#include <rouziclib/rouziclib.h>`
|`rl.c` |Includes `rl.h`, then includes `<rouziclib/rouziclib.c>` exactly once
|Application headers |Include `rl.h` or an application umbrella that includes it
|Application source |Uses the API and is compiled with `rl.c`
|===

== Configuration header

Start with the smallest useful configuration and add optional systems as needed:

[source,c]
----
#ifndef MY_APP_RL_H
#define MY_APP_RL_H

#define LBD 15
#define RL_SDL 2

#include <rouziclib/rouziclib.h>

#endif
----

All rouziclib feature macros must be defined before the umbrella include. See link:configuration.adoc[Configuration] for the important switches and link:optional-backends.adoc[Optional backends] for external dependencies.

== Implementation translation unit

[source,c]
----
#include "rl.h"

#include <rouziclib/rouziclib.c>
----

This pattern makes the implementation see exactly the same feature configuration as the declarations. Include this file in one build target only.

For Objective-C-specific file and dialog helpers, the parallel link:../rouziclib.m[rouziclib.m] amalgamation includes `fileio/dialog.m` and `fileio/file_management.m`. Compile it as Objective-C only when the application needs those implementations.

== A core-only program

For code that only needs the scalar, vector, geometry, memory, text, or math portions, use an `rl.h` without optional backend macros such as `RL_SDL`:

[source,c]
----
#include "rl.h"

int main(void)
{
    // Construct a vector and print its distance from the origin
    xy_t p = xy(3.0, 4.0);
    printf("distance = %.1f\n", hypot_xy(XY0, p));

    return 0;
}
----

Compile the application source and `rl.c` together. For an include such as `<rouziclib/rouziclib.h>`, add the _parent_ of the rouziclib directory to the compiler's include search path. The exact compiler and system libraries depend on the optional features selected.

== SDL application loop

When `RL_SDL` is enabled, `rl_sdl_standard_main_loop()` can own the common SDL event, resize, mouse, keyboard, zoom, cursor, window-manager, and framebuffer-flip work:

[source,c]
----
static void draw_frame(void)
{
    // Draw a world-space line through the active zoom transform
    draw_line_thin(
        sc_xy(xy(-5.0, 0.0)),
        sc_xy(xy(5.0, 0.0)),
        drawing_thickness,
        white,
        blend_add,
        1.0);
}

int main(void)
{
    sdl_main_param_t param = {0};
    param.window_name = "rouziclib demo";
    param.func = draw_frame;
    param.wind_dim = xyi(1280, 720);
    param.show_cursor = 1;

    rl_sdl_standard_main_loop(param);
    sdl_quit_actions();
    return 0;
}
----

On native targets, the standard loop initializes the SDL framebuffer, zoom state, mouse, and fallback or embedded font, and clears/presents the framebuffer after each callback. The callback should draw a frame but must not call `sdl_flip_fb()` itself.

For a custom loop, preprocess input, poll SDL events, process the active text editor, postprocess mouse/zoom state, draw, run `window_manager()`, then flip the framebuffer. Under Emscripten the standard helper executes one frame per call rather than owning a `while` loop, so the application must arrange repeated calls through `emscripten_set_main_loop()` or an equivalent browser loop.

== Build checklist

* Put the parent directory of rouziclib on the compiler's include path when using `<rouziclib/...>` includes.
* Compile the implementation unit once.
* Add system and third-party libraries required by enabled feature macros.
* Add the resource folders used by the application, such as vector font files or image/audio assets.
* For Emscripten, use the `__EMSCRIPTEN__` main-loop path and link the SDL and worker/pthread options appropriate to the target.

The comment at the top of `rouziclib.h` lists the baseline Windows link libraries used by the amalgamation: `winmm`, `comdlg32`, and `ole32` for MinGW/Clang, plus feature-specific libraries selected by the configuration.
