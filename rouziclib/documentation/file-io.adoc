== Raw files, paths, and directories

`memory/generic_buffer.h` provides `buffer_t` helpers for loading and saving raw files. `fileio/path.h` handles filenames, extensions, path joining, application-data directories, and file-kind tests. `fileio/dir.h` loads a directory or a bounded-depth tree into `fs_dir_t`; release it with `free_dir()`.

[source,c]
----
buffer_t source = buf_load_raw_file_dos_conv(path);
if (source.buf != NULL)
{
    // Pass source.buf and source.len to the application parser
}
free_buf(&source);
----

Use the path helpers when constructing paths instead of assuming a platform separator. rouziclib maps directory behavior for Windows, POSIX systems, and reduced WebAssembly builds.

== Images and rasters

`fileio/image.h` loads images into `raster_t` through the built-in loader or an optional library loader. The `IMAGE_USE_*` flags select the desired representation. Common operations are:

* `load_image(path, mode)` for a file
* `load_image_mem(raw_data, size, mode)` for memory data
* `load_mipmap(path, mode)` for tiled/mipmap image data
* `save_image(path, raster, quality)` for non-`.buf` image output
* `convert_image_to_srgb()` and `convert_image_to_frgb()` for format conversion

Keep track of which raster owns its pixel buffer and release it with `free_raster()`. Use the graphics guide for displaying a raster and for colour-space decisions.

== Sound samples

`fileio/sound_format.h` provides sample conversion and loading into 32-bit floating-point buffers. With `RL_SOUND_FILE`, the built-in or selected codec wrappers can decode supported sound formats.

[source,c]
----
size_t sample_count;
int channels, samplerate;
float *samples = load_sound_fl32_select_chan(path, &sample_count, &channels, &samplerate, 0);
if (samples != NULL)
{
    // Pass the samples and samplerate to application DSP code
    free(samples);
}
----

Spacewar uses this path in `game/calc.c` and initializes live SDL audio in `sound.c` with `sdl_audiosys_init(1024)`.

== Fileballs and embedded resources

Fileballs package multiple files into one buffer or generated header. They are useful for distributing fonts and small resource trees without a separate runtime file hierarchy. The core operations are `fileball_make_*()`, `fileball_extract_*()`, and `fileball_find_subfile()`.

Spacewar includes generated bytes from `res_fileballz.h`, wraps them in a `buffer_t`, extracts a `fileball_t`, finds `Spiral.ttf` by name, and passes the subfile data to the TrueType loader. Free the parsed fileball struct when the resource lifetime ends.

== Preferences and dialogs

`fileio/prefs.h` stores simple values in a named preference file. Initialize a `pref_file_t`, get values with `pref_get_double()`, `pref_get_string()`, `pref_get_onoff()`, or `pref_get_2val()`, and persist changes with the matching setter.

[source,c]
----
pref_def = pref_set_file_by_appdata_path("MyApp", "config.txt");
double scale = pref_get_double(&pref_def, "Graphics:Scale", 1.0, "");
pref_set_double(&pref_def, "Graphics:Scale", scale, "");
pref_file_save(&pref_def);
----

Spacewar uses locations such as `Graphics:Drawqueue mode` and `Graphics:Gamma bandaid` and stores its configuration under an application-data path.

File dialogs, system-open helpers, drag-and-drop paths, and Windows message boxes are exposed by `dialog.h`, `open.h`, and the SDL integration header. These APIs are platform-sensitive; guard them when building freestanding or WebAssembly targets.
