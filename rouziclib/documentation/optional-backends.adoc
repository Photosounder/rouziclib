= Optional backends
:toc:
:toclevels: 2
:source-highlighter: highlight.js

Optional integrations are enabled by macros in the application configuration header. They contribute declarations through `rouziclib.h` and implementation code through `rouziclib.c`; the application must also provide the corresponding headers, runtime libraries, and link flags.

== Backend and codec switches

[%header,cols="1,2,3"]
|===
|Define |Library area |Typical reason to enable it
|`RL_SDL` |`libraries/sdl.*` |Windowing, input, audio, framebuffer presentation; value `2` or `3`
|`RL_OPENCL` |`libraries/opencl.*` |OpenCL compute and draw-queue rendering
|`RL_OPENCL_GL` |OpenCL/OpenGL interop |Share the graphics surface with OpenCL
|`RL_VULKAN` |`libraries/vulkan.*` |Vulkan draw-queue backend
|`RL_BUILTIN_GLEW` |Built-in OpenGL extension loader |Use with the OpenGL-backed SDL path
|`RL_CLFFT` |`libraries/clfft.*` |GPU FFT implementation
|`RL_MPFR` |MPFR wrapper and real arithmetic |Extended-precision expression/interpreter operations
|`RL_FFMPEG` |FFmpeg and video/audio helpers |Video decode/encode and FFmpeg-backed audio
|`RL_OPENCV` |OpenCV wrapper |Computer vision operations
|`RL_DEVIL` |DevIL wrapper |Image loading/saving through DevIL
|`RL_LIBSNDFILE` |libsndfile wrapper |Sound-file decoding
|`RL_LIBRAW` |LibRaw wrapper |Camera RAW images
|`RL_LIBJPEG` |libjpeg wrapper |JPEG support
|`RL_LIBCURL` |libcurl wrapper |HTTP and network-backed resources
|`RL_MINIAUDIO` |miniaudio wrapper |Alternative audio implementation
|`RL_OPENAL` |OpenAL wrapper |OpenAL audio output
|`RL_MANYMOUSE` |ManyMouse wrapper |Multi-mouse input
|`RL_TINYEXPR` |tinyexpr wrapper |Expression evaluation
|`RL_FFTPACK` / `RL_CFFTPACK` |FFT implementations |CPU FFT support
|`RL_TRUETYPE` |TrueType loader |Load vector fonts from TTF data
|`RL_IMAGE_FILE` |Built-in image codecs |Common raster image files
|`RL_SOUND_FILE` |Built-in sound codecs |Common sound files
|`RL_ZLIB` / `RL_MINIZ` |Compression |Choose zlib, miniz, or the default cut-down inflate path
|===

Do not define a backend merely because its header is present. Check the wrapper header and source for the platform guard, required link libraries, and the feature-specific initialization call.

Some switches depend on others: `RL_OPENCL_GL` requires `RL_OPENCL` and an OpenGL presentation path, while `RL_CLFFT` requires `RL_OPENCL`. `RL_ZLIB` selects an external zlib implementation; `RL_MINIZ`, `RL_TINYEXPR`, `RL_IMAGE_FILE`, and `RL_SOUND_FILE` select implementations vendored under `libraries/orig`. If neither `RL_ZLIB` nor `RL_MINIZ` is defined, rouziclib keeps its cut-down inflate-only miniz path.

== Spacewar's backend matrix

Spacewar enables SDL2, OpenCL/OpenGL interop, Vulkan, built-in GLEW, FFmpeg, CFFTPACK, miniz, image/sound file support, TrueType, and tinyexpr in its main configuration. Its native Windows build additionally enables CLFFT, LibRaw, and libcurl when compiling with Visual C++.

The build scripts illustrate the external side of this choice:

* `gcc_msys_build.sh` and `clang_msys_build.sh` link SDL, OpenGL, Vulkan, Windows system libraries, and selected codec/compute libraries.
* `embuild_msys.sh` uses Emscripten, SDL2, SIMD, pthreads, a post-JavaScript file, and embedded font resources.
* `build_quick.bat` invokes the Visual Studio solution for the native Windows build.

== Reducing the footprint

Start with the core-only configuration and add one backend at a time. Useful reductions include `RL_EXCL_APPROX`, `RL_EXCL_THREADING`, omitting detailed Unicode data, using the default cut-down miniz path, and avoiding large codec wrappers. `RL_EXCL_THREADING` also removes the built-in audio-bus implementation. `RL_ONLY_DRAW_LRGB` can remove drawing variants when the application never uses floating or queued drawing paths.

== Platform boundaries

SDL, Windows dialogs, GDI screenshots, crash dumps, OpenCL, Vulkan, codecs, and dynamic loading are not equally available on every target. Keep platform-sensitive calls behind the same `#ifdef` boundaries used by the headers and by Spacewar's native versus Emscripten paths.
