== Umbrella and cookbook files

* link:../rouziclib.h[rouziclib.h] assembles the public declarations and applies platform/compiler guards.
* link:../rouziclib.c[rouziclib.c] assembles the implementations.
* link:../rouziclib.m[rouziclib.m] assembles the Objective-C file/dialog helpers.
* link:../quick_reference.h[quick_reference.h] contains short examples for macros, drawing, controls, layouts, input, images, parsing, memory, threading, audio, preferences, and FFTs.
* link:interpreter.adoc[interpreter.adoc] explains expression compilation, execution, decompilation, alternate real types, and WAHE integration.

== Naming patterns

Most APIs follow a predictable family pattern:

* `make_*` constructs a value or state object.
* `load_*` allocates or reads a resource; `free_*` releases it.
* `*_xy`, `*_xyz`, and `*_xyi` select coordinate component types.
* `*_fromlayout` operates on an element from `gui_layout_t`.
* `sc_*` converts world coordinates to screen coordinates; `wc_*` performs the inverse.
* `rl_*` identifies rouziclib's portable threading, atomics, or library-specific helpers.

== Keeping the docs current

When adding a new public module, add its header to the umbrella include list, its implementation to the amalgamated source, a row to this map, and a short usage section in the most relevant thematic guide. Keep feature-specific APIs behind the same macro guards as their headers.
